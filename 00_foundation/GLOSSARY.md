# GLOSSARY

## Purpose
Shared definitions for project terms.
Short, unambiguous, and without long tutorials.

## Terms (v0)
| Term | Definition |
|------|------------|
| Adaptive repetition | Repetition chosen based on an identified weak dimension, not a fixed schedule. |
| Binary gate result | Progression outcome that is only `pass` or `not pass`. |
| Answer key | A complete solution. Provided only on explicit request according to policy. |
| Core concept | A concept that the learner must understand at a basic working level before starting a specific task. |
| Display name | Human-readable name shown in learner-facing and agent-facing material. |
| Phase | Named progression stage that groups related gates under a shared purpose. |
| Preparation | The short pre-task phase where the learner is told what they need to read or understand before starting. |
| Gate | Checkpoint in progression with clear requirements, a stable ID, and a display name. |
| Hint level | Predefined level of help, from direction to (on explicit request) answer key. |
| Readiness check | A short, concept-driven dialogue with the AI that checks whether the learner has a sufficient starting understanding before beginning a task. |
| Curated resource | A resource selected and entered into `RESOURCES.md` with a stable ID. |
| Rubric | Analytical evaluation framework for diagnosing strengths/weaknesses. |
| Sufficient starting level | Enough understanding to begin solving the task productively, without requiring full mastery. |
| Slug | Readable path fragment used after an ID in folder names, such as `cpp-foundations` or `compile-run-basic-io`. |
| Stable ID | Durable identifier used in paths and cross-references, such as `01_phase0`, `gate0`, or `R-001`. |
| Task | Concrete exercise used to practice or evaluate a gate; not the same as the gate itself. |
| Targeted repetition | Repetition chosen for a specific concept or weakness. |
| Strictly controlled teacher | AI role that prioritizes understanding and rules over quick solutions. |
| Responsibility area | Clearly bounded responsibility scope for a file ("one file = one responsibility area"). |
