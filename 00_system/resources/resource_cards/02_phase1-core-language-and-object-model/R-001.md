# R-001 Card for Phase 1

## Resource
- ID: `R-001`
- Title: LearnCpp.com - "Learn C++" tutorials
- Role in repo: Primary reading path for `Phase 1: Core Language and Object Model`
- Best use: First explanation whenever a planned Phase 1 concept is covered well enough for learner-facing use

## Phase 1 Packet A: Numeric Safety Fundamentals
Read in this order:

| Step | Read this | Direct link | Why this step matters |
|------|-----------|-------------|------------------------|
| 1 | Signed integers | https://www.learncpp.com/cpp-tutorial/signed-integers/ | Gives the basic range and overflow model for ordinary integers |
| 2 | Unsigned integers, and why to avoid them | https://www.learncpp.com/cpp-tutorial/unsigned-integers-and-why-to-avoid-them/ | Teaches why unsigned arithmetic often creates surprises instead of safety |
| 3 | Floating point numbers | https://www.learncpp.com/cpp-tutorial/floating-point-numbers/ | Builds the mental model for precision, rounding, and representation limits |
| 4 | Narrowing conversions, list initialization, and constexpr initializers | https://www.learncpp.com/cpp-tutorial/narrowing-conversions-list-initialization-and-constexpr-initializers/ | Connects type ranges to narrowing and gives a first safe-default rule through brace initialization |
| 5 | Arithmetic conversions | https://www.learncpp.com/cpp-tutorial/arithmetic-conversions/ | Explains how mixed-type arithmetic picks a common type |
| 6 | Relational operators and floating point comparisons | https://www.learncpp.com/cpp-tutorial/relational-operators-and-floating-point-comparisons/ | Gives the first workable rule for comparing calculated floating-point values |

## Phase 1 Packet D: Special Members and Dynamic Object Model
Read in this order:

| Step | Read this | Direct link | Why this step matters |
|------|-----------|-------------|------------------------|
| 1 | Introduction to the copy constructor | https://www.learncpp.com/cpp-tutorial/introduction-to-the-copy-constructor/ | Gives the first real model for what happens when one class object is initialized from another |
| 2 | Class initialization and copy elision | https://www.learncpp.com/cpp-tutorial/class-initialization-and-copy-elision/ | Prevents the learner from depending on copy-constructor side effects |
| 3 | Shallow vs deep copying | https://www.learncpp.com/cpp-tutorial/shallow-vs-deep-copying/ | Establishes the safety boundary around resource ownership and copying |
| 4 | The hidden `this` pointer and member function chaining | https://www.learncpp.com/cpp-tutorial/the-hidden-this-pointer-and-member-function-chaining/ | Makes member calls and object-as-implicit-parameter reasoning concrete |

| 6 | Basic inheritance in C++ | https://www.learncpp.com/cpp-tutorial/basic-inheritance-in-c/ | Gives the first practical mechanics of accessing the base and derived surfaces |
| 7 | Virtual functions | https://www.learncpp.com/cpp-tutorial/virtual-functions/ | Adds the first dynamic-dispatch mental model |
| 8 | Virtual destructors, virtual assignment, and overriding virtualization | https://www.learncpp.com/cpp-tutorial/virtual-destructors-virtual-assignment-and-overriding-virtualization/ | Introduces the first safety rule for polymorphic bases |
| 9 | Introduction to operator overloading | https://www.learncpp.com/cpp-tutorial/introduction-to-operator-overloading/ | Gives the rule that operators are just functions with semantic expectations |
| 10 | Overloading the parenthesis operator | https://www.learncpp.com/cpp-tutorial/overloading-the-parenthesis-operator/ | Gives the first callable-object bridge without forcing STL algorithms first |
