# R-006

## Purpose
This card gives a narrow learner-facing packet for Phase 1 lambda basics.

Use it when lambdas belong in `Phase 1: Core Language and Object Model`, but you want to avoid pulling in containers, iterators, or algorithms as hidden prerequisites.

## Phase 1 Packet: Lambda Basics Without STL Prerequisite Drift
Use this packet for:
- lambda syntax as a core language feature
- capture by value versus capture by reference
- the idea that a lambda creates a callable object
- first-pass lifetime caution around reference capture

## Scope Control for Phase 1 Packet
For `Phase 1: Lambda Basics Without STL Prerequisite Drift`, do not continue into generic lambdas, move capture, `this` or `*this` capture, `std::function`, async or threading capture pitfalls, or STL-algorithm-heavy examples unless a later phase explicitly asks for them.

If the learner still needs repetition after this packet, return to the `R-001` lambda lessons only after the gate has explicitly allowed the extra container and algorithm vocabulary those examples often assume.

You are ready to stop when you can explain:
- what the introducer, parameter list, and body of a lambda are
- the difference between capture by value and capture by reference
- why reference capture has lifetime risk
- why a lambda can be treated as a callable object
